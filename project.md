- 주제: 뒤집힌 카드를 클릭하여 같은 그림의 쌍을 찾는 게임
- 꼭 사용해야 하는 기술 스택: p5.js

# 기획

- 윤현준:
    - 전체 UX 플로우 기획
        - 시작 화면 구성: 게임 제목, 설명, 난이도 선택 UI, 시작 버튼
        - 결과 화면 구성: 점수, 시간, 시도 횟수, 다시하기 버튼 UI 동선
    - 연출/인터랙션 아이디어
        - 호버 효과, 클릭 시 애니메이션 줄지? 난이도 선택 시 미리보기 등
        - 지옥/상/중/하 네이밍에 맞는 색감/연출 컨셉
- 방채민:
    - 게임 규칙 고도화
        - 난이도 옵션 정의
        - 점수 시스템 설계
- 손아영
    - 카드 그림 테마 선정: 과일, 커피, 패션 아이템 등
    - 난이도별 카드 디자인 컨셉?
    - 스토리보드 디테일
        - “카드를 클릭하여 시작하세요”, “짝 성공!”, “실패!” 메시지 문구 및 노출 타이밍 정의

# 개발

- 윤현준
    - 화면 전환 및 UI 구현
        - 시작 화면, 난이도 선택 UI 구현
        - 결과 화면 UI 구현
    - 카드 렌더링 & 클릭 처리
        - p5 캔버스에 카드 그리기
        - 마우스 클릭 → 어떤 카드인지 판별하는 로직
        - 방채민이 만든 비교/판정 로직과 연동
    - 애니메이션 연출
        - 카드 뒤집기 애니메이션(sacle/rotate/fill 바뀌는 효과)
        - 짝 성공/실패 시 텍스트/작은 이펙트
    - 손아영
        - 카드 데이터 & 이미지 연결
        - 결과 화면에서 점수, 시간 받아와서 출력하는 함수 구현
        - 다시하기 버튼 클릭시 resetGame 부분 구현
        - UI 텍스트 / 메시지 출력
            - 짝 성공! 다시 도전! 남은 카드: X 쌍 같은 문구
- 방채민
    - 카드 데이터 구조 설계(카드 쌍 매핑, 섞기 로직)
    - 상태 관리
        - 현재 열러 있는 카드 2장 비교 로직
        - 맞으면 고정, 틀리면 1초 후 뒤집기
        - 남은 카드 쌍 수 계산
    - 난이도에 따른:
        - 카드 개수 동적 생성
        - 제한 시간, 횟수 제한 적용
    - 타이머 & 점수 시스템 구현

# 디자인

- 윤현준
    - 전체 화면 UI 스타일
        - 배경 색상, 폰트, 버튼 스타일 정의
        - 난이도별 컬러 테마: 예) 하(파스텔), 상(레드), 지옥(검/빨)
- 방채민
    - “게임 규칙이 한눈에 보이도록” 화면 정보 배치 설계:
        - 상단 UI(타이머/점수/남은 쌍) 레이아웃 러프 스케치
    - 카드 레이아웃/칸 간격, 가독성 중심의 와이어프레임 제안
- 손아영
    - 카드 일러스트 / 아이콘 제작
    - 시작 화면 / 결과 화면의 시각 요소:
        - 타이틀 로고, 배경 패턴, 축하 애니메이션 아이디어(별 튀기기, 폭죽 등)
    - 효과음 / 배경음악 선택:
        - 저작권 프리 사운드 리서치
        - 성공/실패/클리어 사운드 추천 및 적용 도움(경로 정리, 파일명 관리)
