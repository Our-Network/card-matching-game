# 카드 매칭 게임 - 발표 대본 (7분, 2인)

---

## 발표자 A (3분 30초): 기술 기반

### 1. 프로젝트 소개 (1분)

안녕하세요. 지금부터 카드 매칭 게임의 기술적인 부분을 설명드리겠습니다.

이 프로젝트는 **p5.js**와 **Vanilla JavaScript**로 만든 카드 매칭 게임입니다.

**총 19개의 JavaScript 파일**로 구성되어 있고, **5개 계층**으로 분리된 구조입니다.

게임 특징으로는 **4단계 난이도**, **특수 카드 시스템**, 그리고 지옥 모드의 **3장 매칭** 규칙이 있습니다.

**히든 카드**는 매칭 시 모든 카드를 1초간 공개해주고, **폭탄 카드**는 2번 클릭하면 즉시 게임 오버됩니다.

---

### 2. p5.js 게임 루프 (1분 30초)

이 프로젝트의 핵심은 **p5.js 게임 루프**입니다.

`setup()` 함수에서 캔버스를 생성하고, `draw()` 함수가 **초당 60번** 자동 호출되면서 화면을 그립니다.

사용자 입력은 `mouseClicked()` 같은 이벤트 핸들러로 처리합니다.

카드 뒤집기 애니메이션은 p5.js의 **변환 함수**를 활용했습니다.

`push()`와 `pop()`으로 변환 상태를 저장/복원하고, `scale(cos(progress * PI), 1)`로 **X축 스케일**을 조절합니다.

**cos 함수**가 1→0→-1→0으로 변하면서 **3D 뒤집기 효과**가 만들어집니다.

---

### 3. 아키텍처 설계 (1분)

화면의 다이어그램을 보시면 **5개 계층**이 색상별로 구분되어 있습니다.

**Presentation**은 main.js, **Rendering**은 화면을 그리는 역할, **Logic**은 게임 규칙, **Core**는 데이터 모델, **Utils**는 공통 유틸리티입니다.

화살표가 모두 **위에서 아래로** 향합니다. 이것이 **단방향 의존성**입니다.

디자인 패턴으로는 **Observer**, **Factory**, **State** 패턴을 적용했습니다.

이제 **OOO**님이 실제 동작 원리를 설명드리겠습니다.

## 발표자 B (3분 30초): 동작 원리

### 4. 게임 동작 원리 (3분 30초)

#### 이벤트 흐름 (1분 30초)

네, 지금부터 게임이 실제로 어떻게 동작하는지 설명드리겠습니다.

화면의 **시퀀스 다이어그램**을 보시면, 사용자가 카드를 클릭하면 **GameManager**가 `emit('match:success')`로 이벤트를 발행합니다.

그러면 main.js가 CardRenderer와 ParticleSystem을 각각 호출합니다.

여기서 핵심은 **GameManager는 렌더링 코드를 전혀 모른다**는 것입니다. 이벤트만 발행하면 됩니다.

이것이 **Observer 패턴**의 장점입니다. 새 효과를 추가하고 싶으면 리스너만 추가하면 됩니다.

#### 카드 클릭 흐름 (1분 30초)

화면의 **흐름도**를 따라가 보겠습니다.

카드를 클릭하면 먼저 **canFlip**으로 뒤집을 수 있는지 확인합니다.

그 다음 **카드 종류** 분기가 있습니다.

**노란색 히든 카드**는 매칭 시 모든 카드를 1초간 공개해주는 **역전 찬스**입니다.

**주황색 폭탄 카드**는 2번 클릭하면 **즉시 게임 오버**됩니다.

일반 카드는 매칭 검사를 해서, 성공하면 **초록색** 경로, 실패하면 **빨간색** 경로로 진행됩니다.

모든 카드를 매칭하면 게임 클리어, 하트가 0이 되면 게임 오버입니다.

#### 마무리 (30초)

정리하면,

- **p5.js**: setup/draw 게임 루프
- **5계층 아키텍처**: 단방향 의존
- **Observer 패턴**: 느슨한 결합
- **특수 카드**: 히든/폭탄/3장매칭
